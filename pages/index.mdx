Optic's open source tools check the accuracy of your OpenAPI specifications and makes them easy to share validated with your consumers.

```bash filename="Track your OpenAPI specifications" copy=false
# login
optic login
# start tacking an OpenAPI spec
optic api add openapi.yml --history-depth 10

# verify the spec is accurate
optic api verify openapi.yml
```

### Keep your OpenAPI specs up-to-date

Optic's CLI is like git but for APIs. Add a new endpoint? Change an existing one? Optic detects differences and helps developers correctly patch the spec in seconds.

```bash filename="Keep your specification up-to-date" copy=false
# capture traffic
optic oas capture openapi.yml https://api.example.com
# verify accuracy
optic oas verify openapi.yml

# add new endpoints
optic oas verify openapi.yml --document
# update existing endpoints
optic oas verify openapi.yml --patch
```

### Test your API Changes

Optic tests for breaking changes and problematic API designs in your CI pipelines so they can be fixed before being shipped to consumers. A visual API changelog is also added to GitHub/GitLab Code Review so everyone on your team understands exactly what API changes are being considered.

```bash filename="Test API changes" copy=false
# diff OpenAPI changes in the current git branch against main
optic diff openapi.yml --base main

# check for breaking changes
optic diff openapi.yml --base main --check
```
